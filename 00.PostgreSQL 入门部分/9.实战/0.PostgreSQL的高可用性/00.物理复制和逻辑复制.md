# 物理复制和逻辑复制
## 相同点
```
    都是基于 WAL 。
```
### 不同点
```
    1：物理复制是也是流复制，核心原理是将主库的 WAL 日志发送给备库。备库接收到 WAL 日志后进行重做。
       逻辑复制也是基于 WAL 日志的，逻辑复制会根据预先设置好的解析规则解析WAL日志。将WAL二进制文件
       解析成一定格式的逻辑变化信息，之后主库将逻辑变化信息发送给备库。

    2：物理复制可以将DDL操作进行复制，，譬如主库上新建表，对已有表进行加减字段都可以复制到备库上。
       逻辑复制主库上的DDL不会复制到备库上去。

    3：物理复制是实例级别的复制。
       逻辑复制是对数据库表级别进行复制。

    4：物理复制备库只能是只读模式。
       逻辑复制备库既可以读也可以写。

    5：物理复制要求大版本必须一致
       逻辑复制可以跨大版本。
```
